Genre Boilerplates (Cloudflare R2 + CDN – brutal detail)
R2 path: buu-ai-game-maker/boilerplates/{slug}/
CDN URL pattern: https://pub-...r2.dev/boilerplates/{slug}/boilerplate.js + template.md
Forge fetch()s both on task start (cached 1h).
Structure per boilerplate (identical skeleton):
boilerplate.js (three.js r169+): Scene + PerspectiveCamera + WebGLRenderer + Clock + requestAnimationFrame loop + window resize + OrbitControls (examples/jsm) + empty gameLoop(delta) stub + entities = [] array
template.md: Locked system prompt (genre rules + win/lose + TDD hooks)
utils/: coords.js, input.js, entity.js (ECS-lite)
example-scene.json: starter objects
Turn-Based Strategy (Hex-Grid) – slug: hex-tbs boilerplate: Custom HexGeometry (redblobgames axial coords), InstancedMesh (grid 64×64), A* pathfinding (pre-included from three-pathfinding), turn manager stub, unit selection raycaster. template.md: “Strict turn-based hex strategy… generate only axial moves, fog-of-war, resource counters.”
2D-in-3D Side-Scrollers – slug: side-scroller boilerplate: OrthographicCamera locked on X/Z, SpriteMaterial + Sprite for characters, simple platformer physics (velocity + gravity in gameLoop), parallax layers via multiple planes. template.md: “2D-in-3D side-scroller… pixel-perfect collision on Y-up plane, jump/double-jump, parallax.”
3D Roguelike Deckbuilders – slug: roguelike-deck boilerplate: 3D grid (InstancedMesh tiles), card meshes (PlaneGeometry + MeshBasicMaterial), deck array + draw logic stub, procedural room generator (binary space partition). template.md: “3D roguelike deckbuilder… card-as-3D-object, turn-based combat, permadeath.”
Arena Dogfighters / Flight Sims – slug: dogfighter boilerplate: Fork of jakobmaier/threejs-flight-sim (quaternion flight physics, throttle/pitch/roll/yaw, simple drag + lift), third-person chase camera, arena boundary box. template.md: “Arena dogfighter… real-time 6DOF flight, energy management, hitscan weapons.”
Base Builders / Colony Sims – slug: base-builder boilerplate: Procedural grid (raycast placement), building snap system, resource manager stub, SimCity-style tile updates (inspired by open simcity-threejs clones). template.md: “Base builder colony sim… grid placement, population growth, resource flow simulation.”
Custom – slug: custom boilerplate: Empty three.js scene + full controls + comment blocks for any genre. template.md: “Pure custom… no pre-assumptions.”
Forge always starts from these exact files + injects human prompt + Pixel assets. Ready for R2 upload.

Game Flow (Buu AI Chat → Playable Three.js Board – Brutal Detail)
User Input Genre dropdown + prompt submitted → Next.js POST /api/game/init{userId, uuidGameId, genreSlug, rawPrompt}.
Jarvis Ingestion Gemini-Flash-Lite parses → spawns OpenClaw War Room + Trello board (12 fixed tasks in locked order: Checker → Forge → Pixel → Checker).
Live SSE Status Frontend subscribes /warroom/{id}/events (OpenClaw native). Dropdown renders real-time: “Checker • 28 Jest tests written”, “Forge • hex-tbs boilerplate loaded”, progress % + ETA.
Strict TDD Pipeline (JSON-RPC handoffs)
Checker (first): outputs tests/game.test.js + Three.js validators (frame-time, collision, memory).
Forge: fetch(R2 boilerplate.js + template.md) → merges prompt → returns complete game.js (scene, loop, entities, controls).
Pixel: generates base64 sprites/UI → returns texture array.
Checker re-runs full suite on combined output. Fail → auto-loop with error injection until 100 % pass.
Bundle & Publish Jarvis zips final game.js + assets + metadata → uploads to R2 games/{userId}/{gameId}/ (public bucket). Returns permanent CDN URL.
Secure Dynamic Load Chat UI creates sandboxed iframe (sandbox="allow-scripts allow-same-origin", src=/game-runner.html). runner.html:
fetch(CDN game.js) → Blob URL
dynamic import(BlobURL) with pre-bundled THREE r169
mounts renderer.domElement inside iframe
postMessage API for parent controls (resize, pause, save-state).
Delivery & Iteration Playable canvas embeds in chat. Jarvis appends 2 suggested prompt buttons. “Publish Board” creates https://buu.ai/boards/{gameId} (public CDN link). Any failure: Jarvis rollback + retry button + full error log in chat.




Game Tokenization Rules, Logic & Flow (Railway Backend Service – Brutal Detail)
Rules (enforced by Forge + Checker in every boilerplate):
Every game.js MUST include: let playerScore = 0; function addPoints(delta) { playerScore += delta; postMessage({type: 'SCORE_UPDATE', score: playerScore, playerId: currentPlayer}); } + end-game trigger.
Score sent ONLY via postMessage to parent iframe (no localStorage, no external calls).
Leaderboard active ONLY while bonding curve < 100k mcap.
Auto-distribute: exactly 2% of total supply split equally among top-10 unique players at bond-complete snapshot.
Backend Service (new Railway Node.js + Postgres instance):
Table schema: games(id, owner, tokenAddress, supply, mcap, bondStatus, leaderboard JSONB, snapshot JSONB).
Endpoints: /token/launch, /score/update, /leaderboard/sse, /bond/complete.
All writes signed with game-specific JWT (verified in iframe postMessage).
Exact Flow:
User clicks “Publish Board” → game uploaded to R2 → “Launch Token” button appears.
Modal: user enters token name/symbol + chooses initial buy amount (0.1–10 ETH). Jarvis auto-generates fair-launch params.
POST /token/launch → creates bonding-curve contract (custom simple-curve on Railway-managed wallet) + stores tokenAddress + initial supply in Postgres.
Public board page loads: three.js iframe + real-time SSE /leaderboard/{gameId} (updates every 2s from Postgres).
While mcap < 100k: any player score via postMessage → backend validates signature → upserts leaderboard JSONB → broadcasts SSE to all viewers.
Bonding curve auto-monitors buys (webhook from contract). On reaching 100k mcap:
Backend: snapshot top-10 (sorted by score, deduped by wallet if connected).
Migrate: create LP on Uniswap V2/V3, add full liquidity, BURN LP tokens.
Distribute: transfer exactly 2% supply to top-10 wallets (on-chain tx).
Set bondStatus: 'COMPLETE', stop SSE updates, freeze leaderboard forever.
All states immutable in Postgres + on-chain events for audit.
Security:
Score updates rate-limited 1/s per player + signature check.
No user wallet required until token claim.
Leaderboard public read-only SSE; writes only from verified iframe origin.
